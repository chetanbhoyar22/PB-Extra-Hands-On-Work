Decision Making

1. if

When we want to take 1 conditions then we are going to prefer if condition.

2. if...else

When we want to take 2 conditions then we are going to prefer if-else condition.

3. if...else...if

When we want to take more than 2 conditions then we are going to prefer if-else-if condition.

4. switch statement

When we want to take more than 3 conditions then we are going to prefer switch-case condition. & without break statement , it prints all statements.

Q 1 : Explain the difference between var let and const , explain with example?

1. var
var is a Global or Functional Scoped.
var can be reassigned.
var can be redeclare.
var can be modified later.

Example -

            var a=10;
            var a=20;
            console.log(a); //It will assign last value of 'a' variable
            
            Output - 20
2. let
let is a Block Scoped.
let cannot be reassigned.
let cannot be redeclare.
let can be modified later.

Example -
            let a=10;
            let a=20;      //We cannot redeclare
            console.log(a);

            Output - Error 

            let a=10;    
            a=20;
            console.log(a);   //we can modify like this

            Output - 20
3. const
const is a Block Scoped.
const cannot be reassigned.
const cannot be redeclare.
const cannot be modified later.

Example -
            const a=10;
            a=20;
            console.log(a);  

            Output - Throws error that assigned to fixed value, cannot modified.

Q 2 : Explain JS execution?

Javascript execution context is divided into two parts -

1. Creation Phase
2. Execution Phase

1.Creation Phase -
This is the initial phase of Global execution content.Here all variables and functions are given memory allocation ie. stored in heap memory and values assigned as undefined.The global execution context is pushed into the callstack and it remains there until all the code gets executed.After complete execution global execution context gets popped from callstack.

2.Execution Phase -
This is the final phase of global execution context.Here code is executed line by line and the original values are assigned to the variables and to the functions at this stage. When function gets executed a seperate execution context gets executed ie. function execution which is pushed into the stack ie. at the top of the global execution context.Here the code inside the function gets excuted. After the execution of the function the function execution context gets popped out of the callstack and the rest of the code gets excuted.

Q 4 : Use switch case and create Week calendar Monday - print 0 , Tuesday - print 1 and so on Sunday - print 6?

        var day = 6;
        switch  (day) {
            case 0: console.log("Monday - Day - 0");
                    break;
            case 1: console.log("Tuesday - Day - 1");
                    break;
            case 2: console.log("Wednesday - Day - 2");
                    break;
            case 3: console.log("Thursday - Day - 3");
                    break;
            case 4: console.log("Friday - Day - 4");
                    break;
            case 5: console.log("Saturday - Day - 5");
                    break;
            case 6: console.log("Sunday - Day - 6");
                    break;
            default : console.log("OFF - Holiday");
                    break;
        }

        Output - Sunday - Day - 6

Q 5 : Explain Hoisting?

Hoisting - Hoisting is a default Javascript Mechanism where our variables and Functions declarations are moved to top of their Scope before code execution (), only declarations are hoisted not initializations.
